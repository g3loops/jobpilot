# JobPilot — AI Job Matcher & Résumé Builder 🚀

**From a 47 GB job-postings dump → a personalized job-matching engine with AI-tailored résumés and cover letters.**

JobPilot ingests ~50k job postings from a massive public dump (plus live Adzuna postings),
matches them to a candidate's profile using sentence embeddings, ranks them through a
multi-stage scorer that **learns from accept/reject feedback in real time**, and generates a
tailored résumé + cover letter — all behind a clean web UI with built-in analytics.

> Built as an end-to-end big-data application: streaming ingestion, deduplication at scale,
> semantic search, online learning, and an LLM generation layer — deployed as a containerized
> web service.

## 📊 What it does

- **Semantic job matching** — embeds a candidate profile and retrieves the most relevant of ~50k postings using cosine k-NN over 384-dimensional MiniLM vectors (not just keyword search).
- **Multi-stage ranking** — hard filters (location, seniority, contract, visa sponsorship, etc.) + a weighted scorer, with `P@k` / `NDCG` quality metrics.
- **Online learning** — an `AdaptiveLearner` updates ranking weights from accept/reject feedback using reward learning with ε-greedy exploration.
- **AI résumé & cover letter** — tailors a candidate's real experience to a specific job (never inventing experience); falls back to a fully offline template generator when no LLM key is present.
- **ATS scoring** — checks résumé keyword coverage against the job description.
- **Live job feed** — optional Adzuna API pull for fresh postings.
- **Analytics** — top skills, salary distribution, and demand by location/source, including a Count-Min Sketch benchmark.

## 🧠 How it works (architecture)

```
47 GB raw dump ──► byte-offset sampler ──► normalize to canonical schema
                                              │
                        live Adzuna pull ─────┤
                                              ▼
                         streaming ingest ──► Bloom-filter dedup ──► feature enrichment
                                              │
                                              ▼
                    MiniLM embeddings ──► cosine k-NN retrieval ──► multi-stage ranker
                                              │                          ▲
                                              ▼                          │
                              web UI  ◄──►  online feedback learning ────┘
                                              │
                                              ▼
                       LLM (or template) résumé + cover-letter generation
```

Key engineering choices:
- **Scale:** samples a 3.47M-record, 47 GB dump via byte-offset seeking (never loads it into memory).
- **Deduplication:** a sized **Bloom filter** removes duplicate postings in the streaming pipeline.
- **Streaming:** supports real **Google Pub/Sub** streaming with an in-process fallback.
- **Retrieval:** `sentence-transformers` (all-MiniLM-L6-v2) + scikit-learn cosine k-NN.
- **Approximate analytics:** a **Count-Min Sketch** benchmarked against exact counts.
- **Explainability:** every ranked job comes with "why it's ranked here" reasons.

## ▶️ Quick start (local, no API keys needed)

```bash
cd code
pip install -r requirements.txt
uvicorn server:app --port 8000        # -> http://localhost:8000
```

On startup the app loads the job snapshot, the prebuilt embeddings, and the MiniLM model
(~20–30 s), then serves the UI. Open **http://localhost:8000** and walk through
**Matches → Feedback → Analytics**. Pick a built-in test persona to auto-fill a profile, or
upload your own résumé (PDF) to build one.

**No credentials are required.** Optional keys (all in `code/.env`, which is git-ignored):
- `ANTHROPIC_API_KEY` → Claude-Haiku résumé/cover tailoring. Without it, a deterministic
  template backend is used (fully offline).
- `ADZUNA_APP_ID` / `ADZUNA_APP_KEY` → live-postings fetch. Without them, the app runs on the
  shipped snapshot.

See `code/.env.example` for the template.

## 🗂️ Data

The large data files are **not committed** (too big for GitHub) but are regenerated by the pipeline:

| File | Size | What it is |
|------|------|------------|
| `techmap-jobs-dump-2021-09.json` | 47 GB | Raw Kaggle job dump (the input the pipeline samples from) |
| `jobs_snapshot.csv` | 155 MB | 50,302 enriched postings — the pipeline's output |
| `artifacts/job_embeddings.npy` | 74 MB | Cached 384-d MiniLM embeddings |

Rebuild them with:
```bash
cd code
python build_index.py --live --target 50000   # sample → stream → dedup → enrich → embed
```

### `jobpilot` module map

| Module | Responsibility |
|--------|----------------|
| `ingest.py` | normalize, `BloomFilter`, `stream_ingest`, 47 GB byte-offset sampler |
| `pubsub_pipeline.py` | Google Pub/Sub streaming |
| `live_sources.py` | Adzuna live API |
| `features.py` | enrich: seniority, years required, contract, visa, company size, salary |
| `embeddings.py` | MiniLM model, embeddings, cosine k-NN index |
| `ranking.py` | hard filters + weighted scorer + quality metrics (P@k, NDCG) |
| `learning.py` | `AdaptiveLearner` — online reward learning + ε-greedy |
| `generate.py` | résumé / cover-letter `.docx` generation (+ ATS keywords) |
| `analytics.py` | top skills, salary/location/source demand, Count-Min Sketch |
| `benchmark.py` | embedding vs. keyword (TF-IDF) A/B across personas |
